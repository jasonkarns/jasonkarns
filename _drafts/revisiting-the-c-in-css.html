While working on another user style, I ran into a peculiarity between Firefox+Stylish and Chrome+Stylish that I hadn't dealt with in a while. An author's rule with lower specificity was overriding my user-style rule (which had higher specificity), but only in Firefox. The answer lies in the cascading rules in CSS.

Most developers are aware of the specificity rules which govern how style conflicts are resolved. (see specificity) However, we all tend to forget how the precedence of different style sheets is determined. Here's a refresher. In ascending order of precedence:

  1. User Agent
  2. User declarations (normal)
  3. Author declarations (normal)
    1. External style sheets
    2. Embedded style declarations
    3. Inline style declarations
  4. Author declarations (important)
    1. External style sheets
    2. Embedded style declarations
    3. Inline style declarations
  5. User declarations (important)

This cascade supersedes any specificity rankings. Thus,
